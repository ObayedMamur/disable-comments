# TC-120 — Disable Avatars Globally via Plugin Settings

## Summary
Verifies that enabling the "Disable Avatars" option in the plugin's Disable tab removes avatar images from comments sections and author bios site-wide. The plugin sets WordPress's native `show_avatars` option to `false`, suppressing all output from `get_avatar()`.

## Prerequisites
- [ ] WordPress site is running
- [ ] Disable Comments plugin is activated
- [ ] Logged in as Administrator
- [ ] At least one published Post with existing comments exists (so avatar images would normally render)
- [ ] Avatars are currently enabled (default WordPress state: `show_avatars = 1`)

## Test Data

| Field | Value |
|-------|-------|
| Settings page | `/wp-admin/admin.php?page=disable_comments_settings` |
| Settings tab | Disable (the tab where the avatar toggle lives) |
| Avatar toggle label | "Disable Avatars" |
| WordPress option | `show_avatars` |
| Avatar CSS selector | `.avatar, img.avatar` |
| Test post URL | A post with at least one approved comment |

## Steps

| # | Action | Expected Result |
|---|--------|----------------|
| 1 | Navigate to `/wp-admin/admin.php?page=disable_comments_settings` | Settings page loads; navigate to the "Disable" tab if tabs are present |
| 2 | Locate the "Disable Avatars" checkbox/toggle | Toggle is present and currently unchecked (avatars enabled by default) |
| 3 | Navigate to the test post URL on the frontend before saving | Avatar `<img>` elements with class `avatar` are visible next to comments |
| 4 | Return to settings, check "Disable Avatars", and click Save Changes | Settings page reloads with a success notice |
| 5 | Navigate to the test post URL on the frontend | Post with comments loads |
| 6 | Verify that no elements matching `.avatar` or `img.avatar` are rendered in the comments section | Zero avatar image elements are present in the DOM |
| 7 | Scroll to the author bio section (if present) and verify no avatar image is rendered there either | Author profile image is also absent |
| 8 | Reload the settings page and verify "Disable Avatars" is still checked | Setting persists after save |

## Expected Results
- After enabling "Disable Avatars" and saving, the WordPress option `show_avatars` is set to `false`.
- All avatar `<img>` tags (generated by `get_avatar()`) are removed from the rendered HTML.
- The comment author sections show text-only — no profile images.
- The setting persists across page reloads.

## Negative / Edge Cases
- Gravatar or avatar images loaded via JavaScript (not `get_avatar()`) may not be suppressed — only server-side `get_avatar()` calls are affected.
- Some themes may use custom avatar implementations that do not respect `show_avatars`; this is a theme-level issue, not a plugin bug.
- If a caching plugin is active, a cache purge may be needed to see the change.

## Playwright Notes
**Page URL:** `/wp-admin/admin.php?page=disable_comments_settings` (settings), then test post URL (frontend)

**Key Selectors:**
- `#disable_avatars` or `[name="disable_comments_options[disable_avatars]"]` — the "Disable Avatars" checkbox (verify exact name in source)
- `.avatar, img.avatar` — avatar images in comments
- `.comment-author-avatar img, .vcard img` — alternative avatar selectors depending on theme
- `#submit, input[type="submit"]` — Save Changes button
- `.notice-success, #setting-error-settings_updated` — success notice

**Implementation hints:**
- Before saving, take a count of avatar elements: `const before = await page.locator('img.avatar').count()` — should be > 0.
- After saving and navigating to the post: `expect(page.locator('img.avatar')).toHaveCount(0)`.
- The exact checkbox `name` attribute may differ; inspect the DOM or source to confirm (`disable_comments_options[disable_avatars]` or similar).
- This setting uses the standard WordPress `show_avatars` option, not a custom plugin option — the plugin writes it via `update_option('show_avatars', false)`.
